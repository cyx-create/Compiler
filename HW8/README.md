# HW 8

## 背景

本次作业旨在实现一个经典的编译优化算法——**稀疏条件常量传播（Sparse Conditional Constant Propagation，SCCP）**。 SCCP 是一种同时处理**常量传播**与**不可达块消除**的强力优化算法。其核心思想是：不孤立地分析常量，而是将"当前哪些基本块是可能被执行的"与"某个临时变量是否在运行时总是同一个常量"联合求解。这样可以发现更多的常量，并精确地消除死分支与死代码。

## 你需要实现的内容

请在 `lib/opt/opt.cc` 中实现以下函数。框架代码中已给出函数签名。

### 第一部分：`Opt::calculateBT()`

**目标**：通过不动点迭代分析，同步确定：
- 每个基本块是否可能被执行（`block_executable`）
- 每个 SSA 临时变量的运行时值性质（`temp_value`）

你需要根据程序的控制流和数据流，反复迭代直到两个数据结构不再变化。

---

### 第二部分：`Opt::modifyFunc()`

**目标**：基于 `calculateBT()` 的分析结果，重写函数体，产生优化后的 `QuadFuncDecl`。

主要优化包括：
- 删除不可达的基本块
- 删除无用的赋值（即结果为常量的赋值）
- 将已知的常量直接替换到使用处
- 简化条件分支

Phi 节点的处理尤为关键——你需要考虑如何在不可达前驱被删除后，仍然保持 Phi 节点的正确性。

---

### 第三部分：`optProg(QuadProgram* prog)`

**目标**：对程序中的每个函数应用优化，返回优化后的完整程序。

框架已提供 `optFunc()` 的实现骨架，你只需确保上述两个核心函数正确即可。
